# MORE Labs

The MORE section provides deeper explorations of individual Foundry Observability capabilities. Each lab builds on the CORE foundation and can be completed independently.

## Labs

| Lab | Topic | What You'll Learn |
|-----|-------|-------------------|
| [Datasets](./lab-datasets.md) | Test dataset curation & versioning | Create datasets from traces, version them, iterate |
| [Evaluators](./lab-evaluators.md) | Custom evaluators & metrics | Build evaluators beyond built-ins, understand scoring |
| [Continuous Eval](./lab-continuous-eval.md) | Continuous evaluation & monitoring | Set up ongoing eval with trending and alerts |
| [Red-Teaming](./lab-red-teaming.md) | Adaptive adversarial testing | Probe agent safety with adaptive attacks |
| [Tracing](./lab-tracing.md) | Trace-linked analysis | Debug failures through detailed traces |
| [Prompt Optimizer](./lab-prompt-optimizer.md) | Instruction optimization | Systematically improve prompts with the optimization service |

## Scoreboard

Each MORE lab contributes to your scoreboard. As you apply techniques from each lab, re-evaluate your agent and track the improvement. The goal is to see a cumulative uplift across all metrics.

## Suggested Order

While labs can be done in any order, this sequence builds naturally:

1. **Datasets** → Understand what you're evaluating with
2. **Evaluators** → Understand how scores are computed
3. **Continuous Eval** → Automate ongoing monitoring
4. **Prompt Optimizer** → Systematically improve the agent
5. **Tracing** → Debug specific failures in depth
6. **Red-Teaming** → Stress-test safety boundaries
